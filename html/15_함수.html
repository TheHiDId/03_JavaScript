함수(Function, 기능)

    목표하는 결과를 만들기위한 코드의 묶음

    코드를 블록({})으로 묶어서 작성하고 붙여진 이름을 호출해서 사용


    [기본 작성법]

    function 함수명([매개 변수...]) { // 함수 선언
      // 함수 정의
    }

    함수명([전달 인자...]); // 함수 호출

전달 인자(Argument), 매개 변수(Parameter)

    전달 인자

    - 함수 호출 시, () 안에 작성되는 값

    - 작성된 값은 함수의 매개 변수로 전달됨


    매개 변수

    - 함수 선언 시, () 안에 작성되는 변수

    - 호출할 때 전달되는 값(전달 인자)을 저장하기 위한 변수

    - 매개 변수는 함수의 지역 변수

입력받은 수 3개의 합

입력 1: 입력 2: 입력 3:

입력받은 값이 A가 B의 배수가 맞는지 확인

A:
B:

Return(반환)

    함수 내에 작성하는 명령어

    현재 수행 중인 함수를 종료하고 호출한 곳으로 돌아가라는 의미

    - return;, return 값;

    함수 정의 시 리턴을 작성하지 않을 경우 자동으로 함수 정의 끝 부분에 리턴 구문이 추가됨

익명 함수

    이름이 없는 함수 형태

    이름이 없어서 필요한 곳에서 호출하는 것이 불가능

    정의되자마자 실행 또는 변수, 매개 변수, 리턴, 이벤트 핸들러에 주로 사용


    [작성법]

    function([매개 변수...]) {
      // 함수 정의
    }

화살표 함수(Arrow Function)

    익명 함수를 기호(->)를 이용해서 간단히 작성하는 방법
    
    익명 함수: function() {}

    화살표 함수: 
    - () => {}
    - (e) => {}
    - e => {} // 매개 변수가 하나면 소괄호 생략 가능
    - (a, b, c) => {}
    - () => {console.log(123)}
    - () => console.log(123) // 블록 내 내용이 한줄이면 중괄호 생략 가능
    - () => {return 1 + 2;}
    - () => 1 + 2 // 블록 내 내용이 리턴 한줄이면 중괄호와 리턴 생략 가능, 
    - () => [1, 2, 3]
    - () => {return {name: "홍길동"};} // 리턴되는 값이 오브젝트 타입(객체)인 경우 중괄호와 리턴 생략 불가능

function() {}과 () => {} 차이점

    this 참조 변수를 이벤트 핸들러 내부에서 사용 시

    function() {} 의 this === 이벤트가 발생한 요소
    () => {} 의 this ===  window 객체

즉시 실행 함수

    익명 함수가 정의 되자마자 브라우저에 실행되도록 하는 함수


    [작성법]

    (function() {})();

    (() => {})();


    사용 이유

    1. 변수명 중복 문제 해결

      const str = "A"; // 전역 변수
      console.log(str); // A

      (function() {
        const str = "B"; // 블록 내 지역 변수

        console.log(str); // B
      })();

    2. 상대적으로 성능(속도) 우위를 가짐
